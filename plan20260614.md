> 目標：建立一套可維護「上百筆人名、公司名、術語」的詞彙系統，並在選單列提供「開啟詞彙檔」入口。

> 撰寫日期：2026-06-14｜目標方案：approach-6-whisper-macos

## 0. 背景與需求（為何要做）

### 現況痛點
1. 詞彙散落在 `config.json` 四個 mode 各自的 `grok_keyterms` 與 `llm_prompt` 字串中，加一個名字要改多處、容易漏。
2. 人名／公司名可能 **超過 100 筆**，無法塞進：
   - Grok STT 的 `keyterm`（程式已硬上限 `mode.grok_keyterms[:10]`，且 API 對 keyterm 數量有限制）。
   - Cerebras LLM 的 `llm_prompt`（清單一長就 token 暴增、成本上升、LLM 反被干擾）。
3. STT 對中文人名常「**音近字不同**」出錯，例如：
   - `王志明` → `王之名`（下次可能變 `王之明`）
   - 這類錯誤每次形態不固定，純字面 regex 無法窮舉。

### 核心設計決策（已與使用者確認）
- **D1｜新增第三層「拼音 fuzzy 詞典後處理」**：在 LLM 之後、貼上之前，用**無聲調拼音比對**做名字精確替換。此層不呼叫 API、不耗 token、**詞彙數量無上限**。
- **D2｜詞彙集中到單一檔 `user_vocab.json`**：類別化（people / companies / projects / terms），所有 mode 共享。
- **D3｜選單列提供「管理詞彙」子選單**：顯示檔案完整路徑，並提供「用 VSCode 開啟 / 用預設 App 開啟 / 在 Finder 中顯示」三個動作。使用者用外部編輯器自行修改。
- **D4｜熱重載**：以檔案 mtime 偵測變更，每次錄音前自動重載 `user_vocab.json`，**改檔不必重啟程式**。
- **D5｜比對策略 = 無聲調全拼音 + 同字數**：
  - `王志明`(wang zhi ming) == `王之名`(wang zhi ming) == `王之明`(wang zhi ming) → 命中替換。
  - `王大明`(wang da ming) ≠ `王志明` → 不動。
- **D6｜使用者已確認：詞彙表內無「同音同字」衝突項**，因此命中後可直接替換，無需消歧。

### 已知殘留風險（須在程式內留保護開關，見 §5）
- **同音異字的「非詞彙表」真實人名會被誤改**：例如真實存在但不在表中的 `汪志明`(wang zhi ming) 會被改成表中的 `王志明`。
  - 緩解：提供 `match.require_surname_char_same`（預設 `false`）與 `match.use_tone`（預設 `false`）兩個可調參數；並提供 per-term 開關。詳見 §3 schema。

## 3. 檔案規格細節

### 3.1 `user_vocab.json`（新增）

```json
{
  "_comment": "使用者自訂詞彙。改完存檔即可，程式會自動重載，不必重啟。",
  "people":    ["蕭淳云", "周芷萓"],
  "companies": ["加模"],
  "projects":  ["Tahoe"],
  "terms":     ["n8n", "Zeabur", "Zendesk", "Slack", "ERP", "Teams"],
  "overrides": {
    "_comment": "可選：強制字面替換（不經拼音），key=錯誤寫法 value=正確寫法",
    "家模": "加模"
  }
}
```

- `people / companies / projects`：會進入**拼音 fuzzy 比對**（中文人名/公司名主戰場）。
- `terms`：英文/數字術語（如 `n8n`）拼音比對無意義，**僅用於 STT keyterms 與字面比對**，不進拼音引擎。
- `overrides`：字面直接替換，最高優先，給已知固定錯法兜底。

### 3.2 `config.json` 新增區塊

```json
"vocab": {
  "enabled": true,
  "file": "user_vocab.json",
  "stt_keyterm_limit": 10,
  "match": {
    "use_tone": false,
    "require_surname_char_same": false,
    "min_term_len": 2
  }
}
```

- `min_term_len: 2`：長度 1 的詞不做拼音比對（單字誤判率過高）。
- `stt_keyterm_limit`：從 vocab 取前 N 個最重要詞餵給 STT keyterm（與既有 `[:10]` 對齊）。

### 3.3 `_voice_vocab.py`（新增）— 核心引擎

職責與公開介面：

```python
class VocabStore:
    def __init__(self, path: Path, match_cfg: dict): ...
    def maybe_reload(self) -> None:
        """以 mtime 偵測；變更才重新解析並重建拼音索引（熱重載）。"""
    @property
    def stt_keyterms(self) -> list[str]:
        """供 STT 使用的詞（people+companies+projects+terms，截斷到 limit）。"""
    def apply(self, text: str) -> str:
        """第三層主入口：先 overrides 字面替換 → 再拼音 fuzzy 替換。"""

def load_vocab_store(base_dir, config) -> VocabStore | None
```

**拼音索引建立**（`maybe_reload` 內）：
- 對 `people + companies + projects` 每個 canonical 詞，用 `pypinyin.lazy_pinyin(term, style=Style.NORMAL)`（無聲調）得到 pinyin tuple。
- 建 `dict[(len, pinyin_tuple)] -> canonical`。同長度+同拼音的 key 理論上唯一（D6 已確認無同音同字衝突）。

**`apply()` 替換演算法**：
1. `overrides`：對每個 `(wrong, right)` 做 `str.replace`。
2. 拼音 fuzzy：
   - 取所有 canonical 詞的長度集合（通常 2、3、4）。
   - 對每個長度 L（**由大到小**，避免子字串先被吃掉），用滑動視窗掃 `text`：
     - 視窗 substring → 無聲調拼音 tuple。
     - 若 `(L, tuple)` 命中索引，且 substring **本身不等於** canonical（已正確就不動）：
       - 若 `require_surname_char_same` 為真，需首字相同才替換。
       - 記錄替換區間，避免重疊重複替換。
   - 套用所有替換（用區間標記法，從後往前 replace 以保 index 正確）。
3. 回傳新字串。
- **效能**：100 詞 × 短句（通常 < 100 字）滑窗為毫秒級，無虞。
- **保護**：`enabled=false` 或檔案不存在 → `apply()` 原樣回傳，絕不拋例外（比照 Cerebras fallback 哲學）。

### 3.4 `main.py` 改動

1. import：`from _voice_vocab import load_vocab_store`
2. init 區（約第 88 行 llm_correction 之後）：
   ```python
   vocab_store = load_vocab_store(base_dir, config)
   if vocab_store:
       logger.info("   詞彙修正：%d 詞", len(vocab_store.stt_keyterms))
   ```
3. 錄音 callback 開頭：`if vocab_store: vocab_store.maybe_reload()`（熱重載）。
4. pipeline 插入（第 254 行後）：
   ```python
   final_text = normalize_traditional_text(llm_corrected_text, mode)
   if vocab_store:
       vocab_out = vocab_store.apply(final_text)
       if vocab_out != final_text:
           logger.debug("🪵 vocab corrected: %s", vocab_out)
       final_text = vocab_out
   ```
5. session log 增 `vocab_out=`（若採用 §3.6）。
6. **STT keyterm 合併（選配強化）**：目前 STT 只用 `mode.grok_keyterms`。可在 `GrokProvider.transcribe` 把 `vocab_store.stt_keyterms` 併入（去重後 `[:limit]`）。
   - 簡單作法：init 時把 vocab 的詞 merge 進每個 `mode.grok_keyterms`（在 `_voice_config.py` 載入後處理）。**建議採此作法**，改動最小且集中。

### 3.5 `_voice_menubar.py` 改動 — 管理詞彙子選單

在 `_rebuild_menu()` 的 mode 清單之後、結束程式之前插入：

```
管理詞彙 ▶
   📄 user_vocab.json            (disabled，純標題)
   <完整絕對路徑>                 (disabled，灰字顯示路徑)
   ──────────
   用 VSCode 開啟
   用預設 App 開啟
   在 Finder 中顯示
```

- rumps 巢狀選單寫法：`items.append(rumps.MenuItem("管理詞彙"))` 後用 `parent.add(child)`，或用 `self.menu = [..., [parent, [child1, child2, ...]]]` 的 list-of-list 結構。
- disabled 項：建立 `MenuItem` 但**不給 callback**（rumps 中無 callback 即不可點，呈灰色）。
- 三個動作用 `subprocess.run`，全部 try/except 包好：
  - VSCode：`["open", "-a", "Visual Studio Code", path]`（失敗 → log 提示「未安裝 VSCode」，不崩潰）。
  - 預設 App：`["open", path]`
  - Finder 顯示：`["open", "-R", path]`
- menubar 需能拿到 vocab 檔絕對路徑：`build_menubar_app(mode_manager, vocab_path=...)` 多傳一個參數；`vocab_path` 由 main 計算（`base_dir / config["vocab"]["file"]`）後傳入。

## 5. 風險與保護措施

| 風險 | 影響 | 保護 |
|---|---|---|
| 同音異字的非表內真名被誤改（汪志明→王志明） | 改錯字 | `require_surname_char_same` 開關；必要時 per-term 停用；DB 記錄供事後稽核 |
| `pypinyin` 多音字判讀錯誤 | 漏配/誤配 | 用無聲調 NORMAL 風格降低敏感度；多數人名拼音穩定 |
| `user_vocab.json` 格式被使用者改壞 | 載入失敗 | 解析包 try/except，失敗則 log 警告 + 沿用上次成功版本，不崩潰 |
| 熱重載每次錄音讀檔 | 微小 IO | 只比 mtime，未變更不解析 |
| 單字詞誤判 | 大量誤改 | `min_term_len: 2` 過濾 |
| 巢狀選單在 macOS 26 的 rumps 相容性 | 選單異常 | 比照既有 menubar 的 try/except 包覆；失敗不影響核心功能 |
